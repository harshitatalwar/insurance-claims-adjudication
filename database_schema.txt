## ğŸ¯ **Key Design Decisions**

### **1. ID Generation**
- **Policy Holders**: `PH{6-digit-auto-increment}` (e.g., PH000001)
- **Claims**: `CLM{6-digit-auto-increment}` (e.g., CLM000001)
- **Documents**: `DOC{10-char-uuid}` (e.g., DOC4A2B3C4D5E)
- **Dependents**: `DEP{6-digit-auto-increment}` (e.g., DEP000001)

### **2. JSON Fields**
Used for flexible, schema-less data:
- `exclusions`: List of excluded treatments
- `extracted_data`: OCR results (varies by document type)
- `deductions`: Financial breakdown
- `fraud_indicators`: List of fraud flags

### **3. Soft Deletes**
Not implemented - use `status` fields instead:
- Policy Holders: `policy_status` (ACTIVE/INACTIVE)
- Claims: `status` (pending/approved/rejected)
- Documents: `status` (pending/processed/failed)

### **4. Timestamps**
All tables have:
- `created_at`: Record creation (auto)
- `updated_at`: Last modification (auto-update)

```sql
JOIN policy_holders ph ON c.policy_holder_id = ph.policy_holder_id
LEFT JOIN documents d ON c.claim_id = d.claim_id
WHERE c.claim_id = 'CLM000001';
```

### **Check Policy Holder's Remaining Limit**
```sql
SELECT 
  policy_holder_id,
  annual_limit,
  annual_limit_used,
  (annual_limit - annual_limit_used) AS remaining_limit
FROM policy_holders
WHERE policy_holder_id = 'PH000001';
```

### **Get All Claims Needing Manual Review**
```sql
SELECT c.*, mr.reason_for_review
FROM claims c
JOIN manual_reviews mr ON c.claim_id = mr.claim_id
WHERE mr.review_status = 'PENDING';
```

### **Calculate Total API Costs Today**
```sql
SELECT 
  COUNT(*) AS total_requests,
  SUM(total_tokens) AS total_tokens,
  SUM(cost_usd) AS total_cost
FROM api_usage_logs
WHERE DATE(timestamp) = CURRENT_DATE
  AND status = 'success';
```
